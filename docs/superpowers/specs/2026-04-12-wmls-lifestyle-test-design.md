# 五月天粉丝生活方式测试 - 设计方案

## 1. 项目概述

### 项目名称
**WMLS 测试**（五迷生活Style测试）

### 核心定位
一款专为五月天粉丝设计的趣味生活方式测试，通过15-20道融入五月天歌词/场景的题目，测试粉丝在四个维度上的生活态度倾向，最终生成类似MBTI风格的多标签组合型结果。

### 目标用户
- 五月天粉丝群体（WMLS）
- 喜欢参与趣味测试的年轻用户
- 追求社交传播和共鸣表达的用户

---

## 2. 测试维度设计

### 四个核心维度（顺序：W → M → L → S）

| 维度 | 英文标签 | 维度描述 |
|------|----------|----------|
| 工作与生活平衡 | W (Work) | 事业型 vs 生活型 |
| 人生态度 | M (Mindset) | 理想主义 vs 现实主义 |
| 情感模式 | L (Love) | 理性独立 vs 感性依赖 |
| 面对挫折 | S (Struggle) | 倔强坚持 vs 适时放手 |

### 维度标签体系
- **W**: W1=工作狂 / W2=平衡者 / W3=生活家
- **M**: M1=梦想家 / M2=务实派 / M3=乐天派
- **L**: L1=独行者 / L2=追寻者 / L3=守护者
- **S**: S1=不屈者 / S2=弹性派 / S3=释然者

## 3. 题目设计

### 题目风格
- 融入五月天歌词作为题干或选项元素
- 生活场景化题目，让粉丝有代入感
- 无绝对正误，只测偏好

### 题目数量
- **15-20道题**
- 每道题对应一个或多个维度的得分

### 题目示例框架

**例题1（工作与生活 + 人生态度）：**
> "不想上班" 这首歌唱出你的心声，但突然收到老板的加班通知，你会？
> A. "再给我多一点点的勇气"（发完疯还是去加班）
> B. 直接回复"不约"（后果自负）
> C. 默默去加班，心里想着"我心中尚未崩坏的地方"

**例题2（情感模式）：**
> 如果你是《温柔》的主角，对方说"不打扰是我的温柔"，你会？
> A. 乖乖放手，让对方自由飞翔
> B. 倔强地说"为什么不让我打扰"
> C. 默默关注，假装自己已经释怀

## 4. 结果设计

### 结果格式：五迷专属标签型

采用**WMLS谐音梗 + 有趣标签**的组合，每种结果都有独特的"五迷专属人设"称号，让粉丝觉得"这就是我"！

### 核心命名规则
- **W** = 五迷工作观（Work）
- **M** = 五迷人生观（Mind）
- **L** = 五迷情感观（Love）
- **S** = 五迷挫折观（Struggle）

### 结果结构
每个测试结果由**4个字母组合**对应一个**完整人设标签**，例如：
```
W2M1L1S1 = 「加班也要看演唱会的打工诗人」
```

### 人设标签命名风格
- **"加班也要看演唱会的打工诗人"** - 工作是生存，演唱会是信仰
- **"倔强到骨里的佛系五迷"** - 嘴上说躺平，演唱会必到
- **"人间清醒·偶尔发疯·五迷版"** - 正常时很酷，发疯时很真
- **"一个人的演唱会观众席长""** - 独处不孤单，现场最嗨
- **"为演唱会存钱的攒钱大师"** - 其他都可以省，票钱不能动

### 每个维度的描述结构
1. **单字标签**（如"够用就好"）- 10字内
2. **该倾向的具体表现**（2-3句话，带梗）
3. **对应的五月天歌词**（1句）

## 5. 技术架构

### 技术选型
- **前端**：Vue3 + Vite（现代化前端框架，开发体验好）
- **后端**：Express.js（轻量级Node.js后端）
- **数据库**：SQLite（轻量级，无需额外服务）
- **部署**：可打包成Docker容器或直接部署到VPS

### 核心功能
1. **测试答题** - 前端展示题目，用户选择答案
2. **结果计算** - 后端计算四个维度的得分
3. **结果存储** - 记录每次测试结果（可选）
4. **数据统计** - 后台可查看测试数据（管理员功能）
5. **分享功能** - 生成结果图片/链接

### API 设计

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/questions` | GET | 获取所有题目 |
| `/api/submit` | POST | 提交答案，获取结果 |
| `/api/result/:id` | GET | 根据ID获取结果详情 |
| `/api/stats` | GET | 获取统计数据（管理） |

### 数据模型
```javascript
// Question
{
  id: number,
  content: string,        // 题干
  options: [
    { key: string, content: string, scores: { w: number, l: number, m: number, s: number } }
  ]
}

// Result
{
  id: string,
  answers: number[],     // 用户答案
  scores: { w: number, l: number, m: number, s: number },
  resultType: string,    // 如 "W2L1M1S1"
  createdAt: datetime
}
```

## 6. 页面设计

### 页面流程
1. **首页** - 测试介绍 + 五月天氛围图 + 开始按钮
2. **答题页** - 题目卡片 + 选项 + 进度条（SBTI风格）
3. **结果页** - WMLS人设标签 + 百分比展示 + 歌词总结 + 分享按钮
4. **管理后台**（可选）- 查看统计数据

### 页面参考
参考 SBTI / MBTI 测试的界面风格：
- **SBTI 风格**：简洁卡片式、进度条、可爱情绪化
- **MBTI 风格**：专业感、四宫格展示、雷达图

### 五月天风格 UI 元素
- **主色调**：五月天标志性蓝色（#00A8E8）为主，辅以演唱会荧光棒元素
- **字体**：活泼但可读性好
- **动效**：题目切换时有平滑过渡，结果展示时有"揭幕"感
- **背景**：可加入演唱会现场氛围感元素（但不喧宾夺主）
- **按钮**：参考 SBTI 的大按钮、圆润设计

### 视觉风格
- 简洁、年轻化、有共鸣感
- 融入五月天元素（歌词卡片风格、演唱会长曝光感）
- 适合朋友圈分享的视觉设计
